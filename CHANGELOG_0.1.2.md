# rtdata SDK 变更摘要（0.1.2）

## 本次目标
修复异常断线后的自动重连稳定性问题，避免长时间重试后指数退避计算溢出，导致 SDK 自身线程崩溃。

## 核心变更

### 1. 修复重连退避溢出
- 修复 `_connection.py` 中自动重连的指数退避实现
- 不再使用 `2 ** attempt` 这种会随着重试次数无限膨胀的计算方式
- 改为逐次翻倍并受 `max_delay` 上限约束
- 避免在长时间断线、重复 discovery 失败或服务端主动断开后触发 `OverflowError`

### 2. 保持自动重连行为稳定
- 仍然保留抖动退避，避免多个客户端同时瞬时重连
- 最大重连等待时间仍然限制在 30 秒
- 不影响现有认证、订阅恢复和 symbol map 缓存逻辑

## 版本变更
- 版本：`0.1.1` → `0.1.2`

## 交付包
- `dist/rtdata-0.1.2-py3-none-any.whl`
- `dist/rtdata-0.1.2.tar.gz`

## 对客户的实际影响
- 断线后的 SDK 不会因为重连次数过多而自己崩溃退出
- 面对服务端断开、网络抖动、重复 discovery 冲突时更稳定
- 对现有 API 和业务代码无破坏性变更
