# rtdata SDK 变更摘要（0.1.1）

## 本次目标
本次更新主要是为了配合云网关最新的背压和查询可靠性优化，确保 SDK 在“最新值优先”的实时语义下更稳定、更适合交付给客户使用，并避免在日志、文档和示例中直接暴露云网关真实地址。

## 核心变更

### 1. 隐藏网关地址暴露
- SDK 连接日志不再打印真实云网关 IP 和端口，改为通用提示：`Connected to gateway`
- discovery 相关日志不再直接输出 TCP 地址
- 文档和示例中的真实地址改为占位符或改为服务发现模式

### 2. 对齐云网关实时语义
- 明确 SDK 的实时订阅语义是“最新值优先的快照流”
- 不再隐含承诺逐条完整回放每个中间实时更新
- 文档和示例中补充了这一点

### 3. 增强断线与查询失败处理
- 新增 `DisconnectedError`
- 当连接中断时，未完成查询会更快失败，不再只是等待调用侧超时
- history 响应解析增加对显式失败响应的兼容

### 4. 增加异步回调分发
- 新增 `async_callbacks` 和 `callback_queue_size`
- 默认启用异步行情回调分发
- 降低用户慢回调导致 SDK 自己成为慢客户端的概率
- 当本地回调队列积压时，会优先保留更新较新的回调任务

### 5. 文档与交付材料完善
- 更新 `SDK_USAGE.md`
- 更新多个 examples
- 新增 `DELIVERY.md`
- 新增 `README.md`，消除打包时缺少 README 的 warning

## 版本变更
- 版本：`0.1.0` → `0.1.1`

## 交付包
- `dist/rtdata-0.1.1-py3-none-any.whl`
- `dist/rtdata-0.1.1.tar.gz`

## 对客户的实际影响
- 客户不再从日志中直接看到云网关真实地址
- 慢回调场景下 SDK 更稳定
- 断线后查询失败更明确
- 更符合当前云网关“低延迟、最新值优先”的设计目标
